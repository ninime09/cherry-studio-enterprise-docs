## 数据安全与隔离

Cherry Studio 企业版为您的数据安全提供多层保障：

| 安全能力 | 说明 |
| --- | --- |
| **数据隔离** | 基于分组的严格数据隔离，各组织数据完全独立，互不可见 |
| **传输加密** | 全链路 HTTPS 加密 |
| **身份认证** | JWT + OAuth2 认证，密码加密存储 |
| **权限控制** | 基于角色的细粒度权限体系（管理员 / 组管理员 / 成员） |
| **API 安全** | 独立 API Key，按用户隔离调用权限 |

<Callout type="info" title="数据隔离机制">
企业版采用共享基础设施 + 应用层隔离架构。每个组织的数据（模型配置、知识库、助手、文件等）通过严格的分组隔离机制互相隔离，任何 API 请求都会经过身份验证和权限校验，确保只能访问本组织的数据。
</Callout>
