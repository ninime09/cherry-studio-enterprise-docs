## 概述

<Callout type="info">
**入口路径：** 管理后台 → 服务商管理
</Callout>

Cherry Studio 企业版默认集成了 **阿里云百炼 (Aliyun Bailian)** 作为核心模型引擎。系统预置了标准的 OpenAI 兼容协议配置，企业无需繁琐的参数调试，仅需填入 API Key 即可一键解锁通义千问（Qwen）全系列模型及百炼托管的高性能开源模型。

<Callout type="info">
**默认策略与扩展性**

* **默认状态**：系统初始状态下，仅显示"阿里云百炼"这一个服务商。
* **扩展能力**：支持管理员手动添加其他服务商（如 本地 Ollama），实现多渠道模型聚合。
</Callout>

***

## 激活默认服务商

系统列表中已存在 **"阿里云百炼"** 卡片。

<img src="/assets/images/zh/admin/provider-management-aliyun-bailian-qwen-1_zh.webp" alt="" />
1. 点击 **阿里云百炼** 卡片，并点击右上角按钮开启该服务商。
2. **API Key**：填入从阿里云百炼控制台获取的密钥（以 `sk-` 开头）。
3. **API Host**：保持系统默认值。
   * _地址_：`https://dashscope.aliyuncs.com/compatible-mode/v1`
4. 点击 **检查连接**，确保提示"API Key 有效"。
5. 点击 **保存**。

## 管理模型列表

<img src="/assets/images/zh/admin/provider-management-aliyun-bailian-qwen-2_zh.webp" alt="" />
百炼提供的模型列表非常长（如上图所示），为了避免员工选择困难，管理员应进行精简。

1. 点击百炼卡片下的 **𝌡 管理模型**。
2. **筛选策略**：
   * 保留 `qwen3-max` 和 `qwen3.6-plus`（最新一代商业模型）。
   * 保留 `qwq-plus`（用于推理任务）。
   * 保留 `qwen3-coder-plus`（研发专用）。
   * 保留 `qwen3.5-flash`（轻量高频任务）。
   * 按需保留 `qwen3-32b` 或 `qwen3-8b`（开源替代方案）。
3. **隐藏冗余**：将旧版模型（如 `qwen-max`、`qwen-plus`）、带旧日期或过小参数（如 `0.5b`、`1.7b`）的模型点击 **-** 号隐藏。
4. 点击 **保存**。

## 添加其他服务商 (自定义)

虽然百炼是默认供应商，但 Cherry Studio 作为一个聚合平台，完全支持接入其他渠道。

**场景举例：**

* 您想使用 **OpenAI** 原生的 `gpt-4o`。
* 您在本地服务器部署了 **Ollama**。

**操作步骤：**

1. 在服务商列表底部，点击 **+ 添加服务商**。
2. **服务商类型**：
   * 选择 **OpenAI** (通用兼容型)。
   * 选择 **Anthropic** (Claude 系列)。
   * 选择 **Ollama** (本地模型)。
3. 填写对应的 **API Key** 和 **API Host**。
   * _例如本地 Ollama Host_: `http://192.168.1.5:11434/v1`
4. 添加后，该服务商的模型将与百炼模型共同出现在用户的模型选择下拉框中。

## 常见问题 (FAQ)

**Q: `qwen3-max` 和旧版 `qwen-max` 有什么区别？**

A: `qwen3-max` 是当前最新一代旗舰模型（262K 上下文），而 `qwen-max` 指向上一代（Qwen 2.5-Max），仍可用但不再是最优选择。建议新项目直接使用 `qwen3-max`。

---

**Q: `qwq-32b` 和 `qwq-plus` 有什么区别？**

A: `qwq-32b` 是基于开源权重的推理模型，而 `qwq-plus` 是阿里云自研的商业闭源推理模型，通常后者在逻辑上限和安全性上更高，但成本也相对较高。

---

**Q: `qwen-turbo` 还能用吗？**

A: `qwen-turbo` 已停止更新，建议迁移至 `qwen3.5-flash`，后者成本更低、上下文更长（1M）。
