## Quota Hierarchy

Cherry Studio uses a two-level quota system:

1. **Group-level quotas**: Configured in group settings, serving as the default limit for all group members.
2. **User-level quotas**: Per-user custom quotas that override group defaults. Leave empty to inherit the group's default.

## Setting User Quotas

1. Go to the **User Management** page.
2. Click the **Quota** button in the target user's action column.

3. Set limit values for each dimension:

## Viewing Usage and Resetting

Click the expand button next to each quota dimension to view usage details:

<Tabs items={['Express', 'Premium']}>
<Tab value="Express">

<img src="/assets/images/en/admin/user-management-quota-detail-express_en.webp" alt="Express Quota Usage Details" />

* **Last 24 Hours**: Amount consumed in the last 24 hours.

**Note:** When a user reaches their quota, subsequent API calls will be rejected until the quota resets or an admin adjusts it.
</Callout>

## Temporary Quota Adjustment

<Callout type="info">
**Availability:** Currently available in the Express cloud edition. The added amount expires when the period resets and does not modify the original quota configuration.
</Callout>

When a user has an urgent need that exceeds their current quota (for example, the monthly limit is reached but a batch task still needs to run), you can use **Temporary Quota** to inject extra allowance into that user's current period without modifying their regular quota settings.

### Adding Temporary Quota

1. Go to **User Management** and click **Quota** in the target user's action column to open the user quota dialog.
2. Switch to the **Cost Quota** or **Token Quota** tab.
3. Next to the **Daily Limit / Weekly Limit / Monthly Limit** field (beside the chart button), click the **+** button.
4. In the **Add temporary quota — Daily/Weekly/Monthly** popover, enter the amount to add (positive integer; the unit matches the active tab — tokens or currency), then click **Apply**.

<img src="/assets/images/en/admin/user-management-quota-adjust-popover_en.webp" alt="User quota — temporary adjustment" />

<Callout type="warn">
The added amount applies only to the **current period**. It clears together with the period's usage at the next auto-reset and never changes the user's regular quota value. The **Total Limit** field does not support temporary adjustments.
</Callout>

### Viewing Adjustment History

On the **User Management** page, click the **⋯** action menu next to the target user and choose **Adjustment History**.

<img src="/assets/images/en/admin/user-management-quota-adjust-entry_en.webp" alt="Adjustment History entry" />

The dialog lists the user's full adjustment history across all periods in reverse chronological order:

| Column | Description |
|---|---|
| **Time** | Timestamp of the adjustment |
| **Period** | Period the adjustment applies to (Daily / Weekly / Monthly) |
| **Unit** | Unit of the adjustment (Cost / Token) |
| **Amount** | Value added in this adjustment |
| **Operator** | Admin account that performed the adjustment |
