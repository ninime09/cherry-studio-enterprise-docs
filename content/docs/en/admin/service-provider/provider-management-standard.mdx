## Overview

<Callout type="info">
**Navigation:** Admin Console → Provider Management
</Callout>

<img src="/assets/images/en/admin/provider-management-standard-1_en.webp" alt="" />
In this module, admins can configure and manage the AI model sources used by the enterprise. Cherry Studio includes presets for mainstream providers and supports one-click access to OpenAI, Anthropic (Claude), Google (Gemini), as well as OpenAI-compatible domestic relay providers (e.g., SiliconFlow, DeepSeek, OpenRouter).

<Callout type="info">
**Premium edition scope:** Commercial model services accessed via standard API Key and API Host.
</Callout>

## Model Parameters and Pricing

**Action:** Click the **⚙️ Settings** icon next to a model.

To accurately calculate costs, configure pricing per model.

<img src="/assets/images/zh/admin/provider-management-standard-4_zh.webp" alt="" />
### Edit Model Settings

* **Model ID**: The exact identifier used in API calls (must match provider docs).
* **Context length**: Maximum context window (e.g., `32k`, `128k`).
* **Input/Output pricing**:
  * **Input price**: Cost per million input tokens.
  * **Output price**: Cost per million output tokens.
  * **Currency**: `CNY` or `USD`.

<Callout type="warn">
**Why pricing matters:** These unit prices are used by **Usage Analytics**. If left empty, the system cannot calculate costs, resulting in missing financial reports.
</Callout>

## Connectivity Validation

After configuration, validate API Key and connectivity:

<img src="/assets/images/en/admin/provider-management-standard-5_en.webp" alt="" />
1. Click **Check Connection** on the provider row.
2. In the dialog, select a model (e.g., `deepseek-chat`).
3. Click **Confirm**.
4. **Success message:** A toast shows "Check succeeded, API Key valid".

## FAQ

**Q: The provider was added but the model list is empty.**

A: Some OpenAI-compatible providers do not support listing models (`/v1/models`). Click **+ Add Model** at the bottom and enter the Model ID manually (e.g., `qwen-max`).

---

**Q: Why can't my assistant use the knowledge base?**

A: Check whether the assistant's model has the 🔧 (Function Calling) icon. Some lightweight or legacy models do not support tool calls, so knowledge base retrieval won't work.
