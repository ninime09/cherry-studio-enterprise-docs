## Overview

<Callout type="info">
**Navigation:** Admin Console → Providers
</Callout>

<img src="/assets/images/en/admin/provider-management-express-1_en.webp" alt="Express Providers main interface" />

**Cherry Studio Express** is an enterprise-grade, one-stop model aggregation service. Express provides a unified quota pool and direct access to top global models (e.g., Claude Opus 4.6, Claude Sonnet 4.6, Gemini-3.1, GPT-5.4) as well as popular open-source models — no need to apply for API keys from OpenAI, Anthropic, or other upstream providers.

Models are grouped by provider (e.g., AGENT, ANTHROPIC, GOOGLE, OPENAI, QWEN), and each model displays capability tags and input/output pricing.

<Callout type="info">
**Core value**:

* **No ops**: No need to manage dozens of API keys and accounts.
* **Network-free**: Use global models directly within enterprise networks without overseas proxies.
* **Unified billing**: All model usage is deducted from the enterprise balance.
</Callout>

## Balance Dashboard and Top-up

At the top of the page, you can see enterprise usage:

<img src="/assets/images/en/admin/provider-management-express-2_en.webp" alt="Balance dashboard" />

* **Total Balance**: Total funds in the enterprise account. Click "Refresh" to get the latest balance.
* **Used**: Amount and percentage consumed.
* **Available**: Remaining balance available for model calls and percentage.
* **Requests**: Total API call count.

### Recharge

Click the **Recharge** button in the top right, enter an amount or select a preset (¥100, ¥500, ¥1,000, ¥5,000), and choose a payment method to complete the top-up. Balance can be used across all Express models.

<img src="/assets/images/en/admin/provider-management-express-topup_en.webp" alt="Recharge Balance dialog" />

## Model Publishing and Management

The system provides many models by default. Admins can decide which are visible to employees. Three action tabs are available at the bottom of the page: **Manage Models**, **Add Model**, and **Check Connection**.

### Enable/Disable Models

<img src="/assets/images/en/admin/provider-management-express-4_en.webp" alt="Manage Models dialog" />

1. Click **Manage Models** at the bottom.
2. In the pop-up list, browse models grouped by provider. Each model displays capability tags and input/output pricing.
3. Use **+/-** to control model visibility:
   * **+ (Add)**: Add to available list (visible to employees).
   * **- (Remove)**: Hide from employees.
4. Use the search box to quickly find models, or toggle "Selected only" to filter currently enabled models.
5. Click **Save**.

### Manually Add New Models

<img src="/assets/images/en/admin/provider-management-express-5_en.webp" alt="Add Model dialog" />

If a new model (e.g., `gpt-5`) is announced but not yet in the list, you can add it manually:

1. Click **+ Add Model**.
2. Enter the official **Model ID** (e.g., `gpt-5-preview`).
3. Click **Confirm** to use it immediately.

If you want developers to call these Express models directly from external applications, ask them to use the full `model-id@provider_id` format for the `model` field, and refer to [External App Access (Express Models)](/docs/admin/user-management/external-app-access) for the complete setup.

## Check Connection

Before using a model, you can verify connectivity with the provider.

<img src="/assets/images/en/admin/provider-management-express-6_en.webp" alt="Check Connection dialog" />

1. Click **Check Connection** at the bottom.
2. Select the model to check in the pop-up.
3. Click **Confirm** — the system will automatically test model availability.
4. Success feedback: a toast notification **"Check successful, API Key is valid"** appears in the bottom right.

***

## Edit and Delete Models

Admins can edit or delete models that are already listed.

<img src="/assets/images/en/admin/provider-management-express-7_en.webp" alt="Edit Model Settings dialog" />

### Edit a Model

1. Click the **⚙️ Settings** button on the right side of the model to open the "Edit Model Settings" dialog.
2. You can modify the following:
   * **Model ID**: The model's unique identifier (usually no need to change).
   * **Model Name**: Customize the display name.
   * **Model Description**: Add a description for the model.
   * **Capabilities**: Expand to manually specify model capability tags (Vision, Reasoning, Function Calling, Web Search, Agent, Embedding, Rerank).
3. Click **Save**.

<Callout type="warn">
Set capability tags carefully — incorrect configuration may prevent the model from working properly.
</Callout>

### Delete a Model

Click the **🗑️ Delete** button on the right side of the model to remove it from the list.

## FAQ

**Q: Why do I see "Balance Insufficient" even with balance?**

A: The model might be extremely expensive (e.g., preview models), or your enterprise has a daily spending limit.

---

**Q: Why did a model suddenly become gray or disappear?**

A: If an upstream provider (e.g., OpenAI) has a major outage, Express will automatically fuse/disable the service to protect stability. It will return after recovery.
