The **Current License Information** panel shows the license specifications. Understanding these parameters helps capacity planning.

{/* TODO: Add screenshot for authorization parameters */}

| Field | Description | Key Purpose |
| -------------- | --------------------------------- | ---------------------------------------------------- |
| **License ID** | Unique license identifier (UUID). | **Support reference**. Provide this ID when contacting Cherry Studio support to locate your enterprise account. |
| **Activation time** | The date when the license was first imported and activated. | Records go-live time. |
| **Expiration time** | License expiry date. | **Renewal reminder**. The system will usually show a countdown before expiry; renew before this date. |
| **Status** | Current legal validity (e.g., `Valid` / `Expired`). | Determines system availability. |
| **Company name** | The licensed organization name. | Ensures the license holder matches your organization for compliance audits. |
| **User limit** | **Created users / maximum users** (e.g., `6/200`). | **Capacity control**. When user count reaches the limit, **User Management** will block new users. |
| **Licensed features** | Advanced feature package for the current edition (e.g., `express`). | Identifies your product tier (Standard, Enterprise, Flagship, etc.). |

<Callout type="warn">
**Capacity warning:** When the **User limit** bar approaches 90%, contact sales to expand capacity so new employees are not blocked.
</Callout>
