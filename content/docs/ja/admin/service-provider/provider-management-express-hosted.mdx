## 概要

<Callout type="info">
**入口パス：** 管理コンソール → プロバイダー
</Callout>

<img src="/assets/images/ja/admin/provider-management-express-1_ja.webp" alt="Express プロバイダー管理メイン画面" />

**Cherry Studio Express** は企業向けの一括モデル集約サービスです。統一クォータで世界の主要モデル（Claude Opus 4.6、Claude Sonnet 4.6、Gemini-3.1、GPT-5.4 など）や人気 OSS モデルを利用でき、OpenAI/Anthropic 等で API Key を個別に取得する必要はありません。

モデルはプロバイダー別にグループ表示されます（AGENT、ANTHROPIC、GOOGLE、OPENAI、QWEN など）。各モデルには能力タグと入出力価格が表示されます。

<Callout type="info">
**コア価値**：

* **運用不要**：多数の API Key/アカウント管理が不要。
* **ネットワーク不要**：海外プロキシを設定せず、企業内ネットワークで世界のモデルを利用。
* **統一課金**：全モデル利用が企業残高から一括で差し引かれる。
</Callout>

## 残高ダッシュボードとチャージ

ページ上部で資金状況を確認できます：

<img src="/assets/images/ja/admin/provider-management-express-2_ja.webp" alt="残高ダッシュボード" />

* **総残高**：企業アカウントの総残高。「更新」をクリックで最新残高を取得。
* **使用済み**：消費額と比率。
* **利用可能**：モデル呼び出しに使える残額と比率。
* **リクエスト**：API 呼び出しの総数。

### 残高チャージ

右上の **Recharge** ボタンをクリックし、チャージ金額を入力するかプリセット金額（¥100、¥500、¥1,000、¥5,000）を選択して、支払い方法を選んでチャージします。残高は Express の全モデルに使えます。

<img src="/assets/images/ja/admin/provider-management-express-topup_ja.webp" alt="残高チャージダイアログ" />

## モデルの公開と管理

デフォルトで多数のモデルが用意されています。管理者は公開/非公開を決められます。ページ下部に 3 つの操作タブがあります：**モデルを管理**、**モデルを追加**、**接続を確認してください**。

### モデルの有効化/無効化

<img src="/assets/images/ja/admin/provider-management-express-4_ja.webp" alt="モデルを管理ダイアログ" />

1. 一覧下部の **モデルを管理** をクリック。
2. 表示されたリストで、プロバイダー別にモデルを閲覧。各モデルに能力タグと入出力価格が表示されます。
3. **+/-** でモデルの表示/非表示を制御：
   * **+（追加）**：利用可能一覧に追加（従業員に表示）。
   * **-（削除）**：非表示。
4. 検索ボックスでモデルを素早く検索、または「選択のみ」で有効なモデルのみ表示。
5. **保存** をクリック。

### 新しいモデルの手動追加

<img src="/assets/images/ja/admin/provider-management-express-5_ja.webp" alt="モデルを追加ダイアログ" />

Express が新モデル（例：`gpt-5`）を公開したが一覧に未反映の場合は手動で追加可能：

1. **+ モデルを追加** をクリック。
2. 公式の **モデル ID** を入力（例：`gpt-5-preview`）。
3. **確認** をクリックして即時利用。

開発者がこれらの Express モデルを外部アプリから直接呼び出す場合は、`model` フィールドに完全な `model-id@provider_id` 形式を使用し、詳細は <DynamicLink href="/[lang]/docs/admin/user-management/external-app-access">外部アプリ接続（Express モデル）</DynamicLink> を参照してください。

## 接続チェック

モデルを使用する前に、プロバイダーとの接続性を確認できます。

<img src="/assets/images/ja/admin/provider-management-express-6_ja.webp" alt="接続チェックダイアログ" />

1. 下部の **接続を確認してください** ボタンをクリック。
2. ポップアップでチェックするモデルを選択。
3. **確認** をクリック — システムがモデルの可用性を自動テストします。
4. 成功時：右下に **「チェック成功 API Key は有効です」** というトースト通知が表示されます。

## モデルの編集と削除

管理者は公開済みモデルの編集・削除が可能です。

<img src="/assets/images/ja/admin/provider-management-express-7_ja.webp" alt="モデル設定の編集ダイアログ" />

### モデルの編集

1. モデル右側の **⚙️ 設定** ボタンをクリックし、「モデル設定を編集」ダイアログを開きます。
2. 以下の項目を変更できます：
   * **モデル ID**：モデルの一意の識別子（通常は変更不要）。
   * **モデル名**：表示名をカスタマイズ。
   * **モデル説明**：モデルの説明文を追加。
   * **能力**：展開してモデルの能力タグ（ビジョン、推論、関数呼び出し、ウェブ検索、エージェント、埋め込み、リランク）を手動指定。
3. **保存** をクリック。

<Callout type="warn">
能力タグの設定は慎重に行ってください。誤った設定はモデルの正常な動作を妨げる場合があります。
</Callout>

### モデルの削除

モデル右側の **🗑️ 削除** ボタンをクリックすると、一覧から削除されます。

## FAQ

**Q: 残高があるのに "Balance Insufficient" と表示される。**

A: 非常に高価なモデル（プレビュー等）か、企業側で日次上限を設定している可能性があります。

---

**Q: モデルが突然グレーになった/消えた。**

A: 上流プロバイダー（OpenAI 等）の大規模障害時、Express は安定性のため自動停止します。復旧後に自動で再開します。
