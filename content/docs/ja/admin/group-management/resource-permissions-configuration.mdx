## アシスタント権限の設定

Cherry Studio クライアント左側の「アシスタント」欄に表示されるプリセットエージェントを制御します。

* **すべて**：企業ライブラリの全アシスタントにアクセス可能。
* **指定**：**（推奨）** 必要なアシスタントのみを選択し、UI を簡潔に保つ。

**手順：**

1. グループ設定ページの左側で **リソースの可視性** を展開し、**アシスタント** タブをクリック。
2. 上部の **指定** トグルボタンをクリック。
3. リストから対象アシスタントを選択（例：`翻訳アシスタント`、`コードレビュー担当`）。

<img src="/assets/images/ja/admin/group-management-resource-permissions-1_ja.webp" alt="アシスタント権限設定" />

## モデル権限の設定

チャット画面上部のモデル選択で使用できる LLM を制御します。

<Callout type="warn">
**コスト制御のヒント:** モデル権限の設定は API 支出を抑える最も有効な手段です。GPT-4 など高価なモデルはコア部門に限定することを推奨します。
</Callout>

**手順：**

1. 左側の **モデル** タブをクリック。
2. 上部の **指定** トグルボタンをクリック。
3. 設定済みのモデルプロバイダーが一覧表示されます。
4. 以下が可能：
   * **プロバイダー単位で全選択：** プロバイダー名横のチェックで一括許可。
   * **モデル単位で選択：** プロバイダーを展開し Model ID（例：`gpt-5`、`deepseek-coder`）を指定。
5. 検索ボックスでモデル名を検索。
6. **選択済みのみ表示** にチェックして割り当て内容を確認。

<img src="/assets/images/ja/admin/group-management-model-selector-ui_ja.webp" alt="モデルセレクター UI" />

## その他のリソース可視性設定

アシスタントとモデル以外にも、グループ設定ページでは以下のリソースの可視性を制御できます。設定方法は上記と同様です。

<img src="/assets/images/ja/admin/group-settings-tabs_ja.webp" alt="グループ設定の全タブ一覧" />

| タブ | 説明 |
|---|---|
| **サービスプロバイダー** | このグループに表示するモデルプロバイダーを制御。 |
| **アシスタントテンプレート** | リソースライブラリで表示するアシスタントテンプレートを制御。 |
| **エージェント** | このグループに表示するエージェントを制御。 |
| **知識ベース** | 表示する知識ベースリソースを制御。 |
| **ミニプログラム** | 表示するミニプログラムを制御。 |
| **MCP サービス** | 表示する MCP サーバーを制御。 |

<Callout type="info">
各タブは「すべて」と「指定」の 2 モードに対応しています。上記のアシスタント・モデル権限と同様の手順で設定してください。
</Callout>

## クォータ管理

### トークンクォータ

グループ設定ページの左側で **クォータ管理** を展開 → **トークンクォータ** タブをクリックすると、トークンベースの使用量上限を設定できます。

<Tabs items={['Express 版', 'Premium 版']}>
<Tab value="Express 版">

<img src="/assets/images/ja/admin/group-management-token-quota-express_ja.webp" alt="Express 版 トークンクォータ設定" />

</Tab>
<Tab value="Premium 版">

<img src="/assets/images/ja/admin/group-management-token-quota_ja.webp" alt="Premium 版 トークンクォータ設定" />

</Tab>
</Tabs>

トークンクォータは 3 つのレベルで設定します：

**1. グループ総クォータ**

グループ全体の使用量上限を設定します。超過すると全メンバーの使用がブロックされます。

| 設定項目 | 説明 |
|---|---|
| **日次制限** | 1日の使用量上限、翌日自動リセット |
| **週次制限** | 週間の使用量上限、毎週自動リセット |
| **月次制限** | 月間の使用量上限、毎月自動リセット |
| **総制限** | 累計使用量上限、自動リセットなし |

**2. デフォルト個人クォータ**

個別設定のないメンバーに適用されるデフォルト上限です。特定ユーザーに異なる制限を設定する場合は、ユーザー管理で個別に設定できます。

**3. メンバー枠上限（Member Quota Cap）**

グループ管理者がメンバーに設定できるクォータの最大値です。グループ管理者が過大なクォータを割り当てることを防止します。

<Callout type="info">
Token 制限と金額制限は同時に適用されます。いずれかの上限に達すると使用がブロックされます。空欄は無制限を意味します。
</Callout>

### 金額クォータ（Express 版専用）

<Callout type="info">
この機能は **Express 版** でのみ利用可能です。グループ設定ページの左側で **クォータ管理** を展開 → **金額クォータ** タブをクリックしてください。
</Callout>

金額クォータは、Token 数ではなく金額ベースでグループおよびメンバーの API 使用コストを管理します。

<img src="/assets/images/ja/admin/group-management-cost-quota-1_ja.webp" alt="金額クォータ設定" />

トークンクォータと同様に、金額クォータは 3 つのレベルで設定します：

**1. グループ合計クォータ**

グループ全体の金額使用上限です。超過すると全メンバーの使用がブロックされます。日次・週次・月次・合計の 4 次元をサポートします。

**2. デフォルトメンバー割り当て**

個別設定のないメンバーが継承するデフォルト値です。

**3. メンバー枠上限**

グループ管理者がメンバーに設定できるクォータの最大値です。

<Callout type="warn">
Token 制限と金額制限は同時に適用されます。いずれかの次元が上限に達すると、ユーザーの以降の API 呼び出しはブロックされます。
</Callout>

### 一時クォータ加算

<Callout type="info">
**対応バージョン：** 現時点では Express クラウド版で利用可能です。
</Callout>

**クォータ管理** → **金額クォータ** または **トークンクォータ** のページでは、期間型の各上限フィールド（1日 / 週間 / 月間）の右側に **+** 一時加算ボタンがあります。**グループ合計クォータ** または **デフォルトメンバー割り当て** の当期のみに追加枠を注入でき、常設のクォータ値は変更されません。

<img src="/assets/images/ja/admin/group-management-quota-adjust-popover_ja.webp" alt="グループクォータ — 一時加算" />

**操作方法：**

* 上限フィールド右側の **+** ボタンをクリック → ポップオーバーで追加額を入力 → `Apply` をクリック。
* 追加された分は期間の自動リセット時に失効し、保存済みの上限値は変更されません。
* **合計限度** フィールドは一時加算をサポートしません（自動リセット機構がないため）。

**履歴の確認：** ページ右上の `Adjustment History` ボタンをクリックすると、当該グループの全期間にわたる加算履歴ダイアログが開きます。

<img src="/assets/images/ja/admin/group-management-quota-adjust-history_ja.webp" alt="グループ加算履歴" />

## 権限設定の検証

正しく設定されているか確認するため、簡易テストを行います：

1. テストユーザーを作成し新グループに割り当て。
2. テストアカウントでクライアントにログイン。
3. **チェック A:** 新規チャットでモデル一覧を開き、選択したモデル **のみ** が表示されることを確認。
4. **チェック B:** アシスタントライブラリで許可済みのアシスタントのみ表示されることを確認。
